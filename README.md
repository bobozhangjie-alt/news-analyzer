# 新闻偏见检测器（News Bias Detector）

一个用于辅助批判性阅读新闻的前端应用。

用户可以粘贴新闻全文，系统会从**立场倾向**、**偏见句子**、**情绪词汇**和**多角度阅读建议**四个方面给出分析结果，帮助识别潜在的信息偏差与叙事倾向。

---

## 1. 项目目标

在信息茧房和媒体叙事分化日益明显的背景下，本项目旨在提供一个轻量、可解释的新闻阅读辅助工具：

- 帮助用户快速识别文章整体倾向（偏左 / 中立 / 偏右）
- 高亮可疑偏见句子并解释“为什么可疑”
- 标注情绪化词汇与绝对化表达
- 提供可执行的“对立视角补充阅读建议”

> 注意：本工具为规则引擎辅助判断，不是事实裁决器，结果仅供参考。

## 2. 已实现功能

### 2.1 新闻输入

- 支持粘贴新闻全文（中文）
- 支持字数统计
- 支持一键重置

### 2.2 示例文章（4 选 1）

点击“加载示例”可选择以下 4 种样本：

1. 经济政策争议（偏右 · 高情绪）
2. 环境保护报道（偏左 · 中情绪）
3. 移民政策分析（中立 · 低情绪）
4. 社会福利改革（偏左 · 高情绪）

### 2.3 立场分析

- 输出：`偏左 / 轻微偏左 / 中立 / 轻微偏右 / 偏右`
- 展示左右关键词命中数量和可视化占比
- 给出置信度与解释描述

### 2.4 偏见标注（左右双栏联动）

- **左侧**：原文全文，高亮有偏见嫌疑句子
- **右侧**：偏见句子列表（可展开查看原因）
- 支持双向联动：
  - 点击左侧高亮句子，右侧自动定位到对应条目
  - 点击右侧条目，左侧对应句子高亮激活并滚动定位

### 2.5 情绪词汇识别

按三类标注并统计：

- 负面情绪词
- 正面情绪词
- 绝对化/强化词

并给出情绪指数与等级：

- 高度情绪化
- 中度情绪化
- 较为理性

### 2.6 多角度建议

根据立场和情绪状态动态生成建议，如：

- 寻找对立阵营视角
- 寻找中性客观报道
- 核实原始信息来源
- 了解国际媒体视角

## 7. 分析引擎说明（`src/lib/biasAnalyzer.js`）

### 7.1 立场分析（关键词计分）

- 内置左倾关键词词库与右倾关键词词库
- 根据命中频率计算左右比例
- 阈值逻辑：
  - 左侧占比 > 65%：偏左
  - 右侧占比 > 65%：偏右
  - 占比 > 55%：轻微偏向
  - 否则：中立

### 7.2 偏见句检测（规则匹配）

当前规则覆盖：

- 来源可疑（匿名来源、模糊来源）
- 绝对化表述
- 预设立场
- 标签化描述
- 二元对立
- 情绪渲染（单句情绪词过密）

### 7.3 情绪词识别

- 负面词、正面词、强化词分组检测
- 输出每个词的命中次数
- 计算情绪化密度（0~100）并分级

### 7.4 综合偏见指数

综合考虑：

- 偏见句数量
- 情绪密度
- 是否存在明显立场偏向

最终输出：

- 偏见指数（0~100）
- 偏见等级（轻微/中度/高度）

## 8. 交互设计说明（偏见标注）

偏见标注页采用双栏结构：

- 左栏：原文语境，避免“断章取义”
- 右栏：结构化偏见证据，便于逐条复核

联动机制：

- `activeBiasIndex` 作为两侧同步索引
- 左侧点击高亮句 -> 更新索引 -> 右侧 `scrollIntoView`
- 右侧点击条目 -> 更新索引 -> 左侧句子激活高亮

## 9. 可配置与可扩展项

如需提高准确性，可优先从以下方向迭代：

1. 扩充关键词词库（按领域细分：经济、外交、社会议题）
2. 增加偏见规则（如因果倒置、选择性统计、诉诸权威）
3. 增加否定语境处理（减少误报）
4. 引入模型辅助（规则 + LLM 混合判定）
5. 增加“对比多篇新闻同题分析”能力

## 10. 已知限制

- 当前以规则与词库为主，语义理解能力有限
- 对复杂修辞、反讽、跨段上下文的识别仍有提升空间
- 结论不应替代人工判断与多源核验

## 11. FAQ

### Q1：为什么有些句子看起来中性也被标注？

A：规则引擎会优先捕捉“可疑模式”，属于风险提示而非定罪判断，建议结合上下文复核。

### Q2：为什么一篇文章可能同时出现左右倾向词？

A：真实报道常包含多方观点。系统会根据占比和阈值输出“偏向”或“中立”。

### Q3：必须联网吗？

A：当前核心分析逻辑在前端本地规则中执行，不依赖后端接口。
